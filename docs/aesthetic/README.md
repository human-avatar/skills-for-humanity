# ✨ Aesthetic

> *Finding the version that is right, not just finished.*

---

## What this category is

Aesthetic tools apply formal reasoning to questions of form — whether something coheres, whether it's more complex than it needs to be, whether the pattern at work is being used well, and whether a simpler version exists. This isn't taste or opinion; it's structured analysis of how the parts of something relate to each other and to the whole.

The aesthetic toolkit works on any artifact: a product design, a piece of writing, a system architecture, a presentation, an argument. The same principles that distinguish a good building from a bad one distinguish a good API from a bad one.

---

## When to reach for it

- Something feels off but you can't name what
- A design, system, or document was built by multiple contributors and feels inconsistent
- You suspect you've added complexity that isn't earning its place
- You want to understand *why* something works, not just that it does
- You need to strip something back to its essence without losing what matters
- You're about to present or ship something and want a final coherence pass

---

## When *not* to reach for it

**Don't confuse aesthetic with subjective preference.** These tools analyze structure and coherence, not style choices. If the real question is "which of these two valid options do I prefer," that's a decision question, not an aesthetic one.

**Don't use aesthetic tools for functional problems.** If something is broken or incorrect, fix it first. Aesthetic tools work on things that function correctly but could be *better*.

The nearest neighbor that trips people up: [Sensory](../sensory/) — which is about observational precision, not formal analysis of an artifact.
