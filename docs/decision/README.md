# ⚖️ Decision

> *The option space is almost always narrower than it should be.*

---

## What this category is

Decision tools impose structure on choice — expanding the option set, weighting criteria explicitly, stress-testing a direction by imagining it failed, and calibrating how much process a decision actually deserves. They exist because decision-making has well-documented failure modes: false dichotomies, premature commitment, invisible risks, and wasted deliberation on reversible choices.

These tools don't make decisions for you. They make the decision-making process visible so you can own it, rather than have it happen to you.

---

## When to reach for it

- Choosing between options — and not confident you're seeing all of them
- About to commit to a significant direction and want to stress-test it
- Multiple criteria matter and you want to compare options systematically rather than by gut
- Not sure how much time to spend on this decision
- Leaning toward a direction but want to force yourself to see why it might fail
- Decision has multiple stakeholders who weight criteria differently

---

## When *not* to reach for it

**Don't use decision tools when the real problem is unclear options.** If you don't know what you're choosing between, generate options first — that's [Creativity](../creativity/) (specifically [alternatives](../creativity/alternatives.md) or [concept-fan](../creativity/concept-fan.md)).

**Don't use decision tools when the real problem is a values question.** "Should we do this?" is often not a decision question — it's an [identity](../identity/) or [ethics](../ethics/) question. Clarify whether this is a trade-off analysis or a values test before reaching for decision tools.
