# ✍️ Prompt

> *A good prompt doesn't just ask better — it structures how Claude thinks.*

---

## What this category is

The prompt toolkit helps you write better instructions for Claude — applying Anthropic's best practices to take a prompt from vague to precise, from flat to structured, from producing inconsistent results to producing reliable ones. The core insight: prompts fail not because they're too short or too long, but because they leave too much to inference, don't explain the *why* behind instructions, or mix instructions and content without separation.

This category has one skill. It handles the full range from "this prompt isn't working" to "help me write a prompt for X from scratch."

---

## When to reach for it

- A prompt keeps producing results that aren't quite right
- Claude isn't behaving the way the instructions intend
- You want to write instructions for a skill, agent, or system prompt and want them to be effective
- You have a working prompt but know it could be better
- You're about to invest significant time in a Claude workflow and want the foundation to be solid

---

## When *not* to reach for it

**Don't use prompt-elevate when the issue is with the task, not the prompt.** If you're asking Claude to do something it shouldn't do, a better prompt won't fix it.

**Don't use it for conversational exchanges.** Prompt engineering applies to instructions that will be reused or deployed — not to one-off questions in a conversation.
