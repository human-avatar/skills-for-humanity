# 🔋 Resource

> *The system can only move as fast as its slowest point. Everything else is noise.*

---

## What this category is

Resource tools apply structured thinking to the management of limited inputs — time, money, people, attention, capacity. They find what's actually constraining throughput (not what seems to be), identify where the same effort produces the most output, distribute resources across competing needs with explicit trade-offs, and surface where resources are being lost silently.

The unifying principle: most resource problems are invisible until they're acute. These tools make the invisible visible before it becomes a crisis.

---

## When to reach for it

- Things are moving more slowly than they should and you don't know why
- Competing priorities and not enough capacity — need to make allocation decisions explicit
- Sense that effort isn't producing commensurate output
- Want to maximize the impact of what's available without adding more
- A team or project is perpetually behind despite working hard
- Preparing resource allocation decisions that will be contested

---

## When *not* to reach for it

**Don't use resource tools when the problem is a strategy problem.** Optimizing the allocation of resources in service of the wrong goal is efficient waste. Make sure you're working on the right thing before optimizing how you work on it.

**Don't use resource tools as a substitute for having the capacity conversation.** Sometimes the answer to a resource analysis is "we need more resources" — and that's a valid output. Don't let optimization tools be used to avoid that conversation.

The nearest neighbor: [Systems](../systems/) — specifically leverage analysis, which works at the systemic level. Resource tools focus on operational allocation; systems tools focus on structural intervention points.
